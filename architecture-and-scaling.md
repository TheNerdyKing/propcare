# PropCare Tech Stack & Architecture Overview

## Current Tech Stack

### Front-End 
- **Framework**: React 19 + Vite 6
- **Styling**: Tailwind CSS 4 (Utility-first, modern responsive UI)
- **UI Components & Icons**: Lucide React for iconography; `framer-motion` for fluid micro-animations.
- **Form Handling**: React Hook Form with Zod validation.

### Back-End 
- **Runtime**: Node.js (via `tsx` in dev, standard Node in production)
- **Framework**: Express.js
- **Real-Time Engine**: Socket.IO for live property and ticket updates across clients.
- **ORM & Database**: Prisma ORM, currently configured with `better-sqlite3` for local development, easily translatable to PostgreSQL (like Supabase, AWS RDS) for production scale.
- **Document Processing**: `exceljs` for reports, `pdfkit` for document generation. 
- **Email Delivery**: `nodemailer` configured for SMTP dispatch.

### Hosted Infrastructure
- **Deployment**: Vercel (`propcare-api.vercel.app` & frontend), taking advantage of serverless functions and edge networking.

### AI & Integrations 
- **AI Core**: Google Generative AI (`@google/genai`), implementing Gemini 3 Flash to automatically categorize, prioritize, and coordinate emergency procedures for tenant reports.
- **Mapping (Planned)**: Google Maps / Places API integration to automatically fetch top-rated nearby contractors and dynamically route requests based on physical proximity.

## Roadmap: Scaling for External Customers (SaaS Model)

The goal is to scale PropCare effectively for full-time external customers and Property Management aggregators. This requires a shift in authentication and billing paths:

### 1. Future Authentication & Access Architecture
- **Multi-Tenant System**: The database will inherently split users into `Tenants` (Real Estate Firms).
- **Organization-Based Access Roles**: 
  - Admin (Can manage subscriptions & billing, block users)
  - Manager (Can view all properties within the company)
  - Handyman/Contractor (View restricted to assigned tickets only)
  - Tenant (No login required, access only via public complaint portal with unique ticket reference).

### 2. Subscription, Free Trials, and Invoicing Flow
To address the "5 days free trial -> subscription -> invoice or credit card" requirement:
1. **Stripe Integration**: 
   - We will implement **Stripe Billing** to oversee trials and recurring subscriptions automatically. 
2. **Onboarding Sequence**:
   - Agency securely registers -> `Trial Mode` initiates (5 days).
   - After Day 5 -> System prompts the user to select payment:
     - **Credit Card**: Validated instantly via Stripe Elements. Auto-charged monthly.
     - **Invoice**: For enterprise/traditional clients, a Net 30 invoice is auto-generated by Stripe/pdfkit and emailed.
3. **Grace Period & Blocking**: 
   - A background cron job will monitor payment statuses within Stripe. 
   - `Overdue Payment`: Account restricted. They can still log in, but instead of the Dashboard, they face a locked payment gateway to settle immediately. 
   - **Internal Billing Dashboard**: We will build an internal super-admin panel for *your* team to review global payments, see active/pending/overdue accounts, and manually block/unblock accounts.
